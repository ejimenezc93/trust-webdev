2. **Mensualidad de operación administrada** — recurrente.
3. **Mejoras, entrenamiento avanzado o nuevas automatizaciones** — por proyecto, sprint o bolsa de horas.

## Qué debe incluir la mensualidad

La mensualidad no es “por si algo se rompe”. Es operación continua de un sistema AI vivo.

Debe incluir:

- Monitoreo de instancia.
- Actualizaciones de OpenClaw y dependencias cuando aplique.
- Backups.
- Revisión básica de logs/errores.
- Ajustes de prompts y comportamiento.
- Soporte a usuarios.
- Pequeñas mejoras mensuales dentro de un límite.
- Seguridad básica.
- Gestión de consumo AI/API.
- Revisión de costos.
- Mantenimiento de integraciones.

Debe tener límites:

- Horas incluidas al mes.
- Tiempo de respuesta.
- Qué cuenta como bug.
- Qué cuenta como mejora.
- Qué se cotiza aparte.
- Límite de API usage incluido.

## AI/API usage y hosting

Recomendación: cobrar hosting/API separado o incluir un límite claro.

Ejemplo:

- VPS: pagado directo por cliente o facturado aparte.
- API usage: se cobra al costo + 15–25% de gestión, o se incluye hasta un monto mensual.
- Si el consumo pasa el límite, se factura adicional.

Frase:

> La mensualidad cubre administración, soporte y mantenimiento. El consumo de modelos AI y hosting se maneja con límites claros porque depende del volumen real de uso.

## Entrenamiento del AI legal/fiscal

Cuidado con la palabra “entrenar”. Probablemente no necesitan fine-tuning al inicio. Necesitan:

- Knowledge base curada.
- Instrucciones/prompting.
- Procedimientos internos.
- Plantillas.
- Ejemplos aprobados.
- Evaluación humana.
- RAG/document retrieval si aplica.
- Guardrails para no dar asesoría final sin revisión.

Frase útil:

> Cuando hablamos de entrenar el AI, mi recomendación inicial no es hacer fine-tuning desde el primer día. Primero conviene construir una base de conocimiento, criterios de respuesta, plantillas y flujos con revisión humana. Eso es más seguro, más barato y más fácil de mantener.

## Riesgos legales/fiscales

Esto debe quedar clarísimo:

- El AI no debe emitir criterio legal/fiscal final sin revisión profesional.
- El cliente mantiene responsabilidad profesional por decisiones finales.
- El sistema puede asistir, organizar, resumir, comparar, preparar borradores y sugerir estructuras.
- Outputs deben tener revisión humana.
- No prometer exactitud absoluta.
- No prometer reducción fiscal garantizada.

Cláusula conceptual:

> El sistema será una herramienta de apoyo operativo y analítico. No reemplaza el criterio profesional del equipo legal/fiscal ni asume responsabilidad por decisiones tributarias, legales o contables.
